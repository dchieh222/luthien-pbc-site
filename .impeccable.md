## Design Context

### Users
**Primary:** Tech-savvy Claude Code users — engineers who use AI coding agents daily and want control over what the agent can do. They understand proxies, env vars, and terminal workflows.
**Secondary:** Casual Claude Code users — PMs and less technical people who forward to engineers. Copy must work for them too ("plain English first").
**Job to be done:** Evaluate whether Luthien solves the trust/control problem with AI coding agents, then get running in under 2 minutes.

### Brand Personality
**Credible, deadpan, technical.**
- Human-first positioning — Luthien exists because humans should stay in control
- Humor from absurdity, not exclamation points
- Developer solidarity tone ("we get the problem")
- No AI slop: no em dashes, no fabricated UI, no fictional code, no preamble

### Aesthetic Direction
- **Theme:** Dark mode only (`#09090b` base)
- **Accent:** `#6b9fff` (blue) primary, `#22c55e` (green) for security/success
- **Fonts:** Inter (body/headlines) + JetBrains Mono (code/labels)
- **Feel:** Developer-native, terminal-adjacent, minimal but not sterile
- **Anti-references:** Marketing sites with "Book a demo" CTAs, gradient-heavy SaaS landing pages, anything that looks like AI slop
- **Max-width:** 960px container, content-first layout

### Design Principles
1. **Demo-first** — Open with Luthien in action (before/after), then explain. Show, don't tell.
2. **Incremental refinement** — Improve what exists. No wholesale redesigns. Every change should be a clear improvement over the previous state.
3. **Copy integrity** — The site must not exhibit the problems Luthien claims to solve. No em dashes, no fabricated examples, no fictional class names.
4. **Zero friction** — 2 env vars, keep your client. "Try it" not "book a call." De-emphasize install complexity.
5. **Trust through transparency** — Real quotes, real incidents, real code. Security/encryption details visible. No hand-waving.

### Content Requirements
- Demo-first layout with annotated SVG before/after
- Plain English copy that PMs can forward to engineers
- Two-layer story: universal dangers + custom business rules
- Complete block story: what happened, what user sees, what to do next
- Supported clients: Claude Code, Codex, Cursor = YES; Windsurf = NO
- Security/trust: encryption, private code handling, SOC 2 readiness
- Funny, deadpan copy matching Jai's register

### Typography Scale
- **Base:** `html { font-size: 17.6px }` — all `rem` values are relative to this
- **Hero h1:** `clamp(2.2rem, 6vw, 4rem)`
- **Hero subtitle:** `clamp(1.1rem, 2vw, 1.4rem)`
- **Section label** (JetBrains Mono, uppercase, `letter-spacing: 0.18em`): `1.25rem`
- **Section h2 / inline headings:** `clamp(1.2rem, 2.5vw, 1.5rem)`
- **Section header class:** `clamp(1.5rem, 4vw, 2rem)`
- **"without luthien" / "with luthien" demo labels** (JetBrains Mono, `letter-spacing: 0.18em`): `1rem`
- **Body / card text:** `0.95rem`
- **Small / meta text:** `0.85rem`
- **Micro / code labels:** `0.75–0.8rem`
- **Buttons:** `0.95rem`, Inter, weight 600

### Technical Constraints
- Plain HTML/CSS/JS, no build system
- All CSS inline (except blog pages share blog.css)
- Each version directory is self-contained
- GitHub Pages deployment from site/ directory
- Version directories are frozen once on main
