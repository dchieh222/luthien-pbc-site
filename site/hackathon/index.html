Build on Luthien this weekend

Open-source proxy for AI control research. Sits between AI coding agents and model APIs. Enforces policies, logs everything, catches dangerous stuff before it executes.

Jai Dhyani (Luthien co-founder) is a speaker and judge at this hackathon. Catch his talk Friday at noon for a live demo.

Why Luthien for this hackathon? The hackathon calls for "control protocols that involve traditional security measures, such as giving the trusted model access to a firewall." Luthien IS that firewall. It sits between agent and API, runs policy checks (regex, DLP, LLM-as-judge), and logs everything. You get the interception layer for free. Focus on the research.

Project Ideas

Each idea maps to a hackathon track and uses Luthien as real infrastructure. Pick one, remix one, or bring your own.

Proxy-Level Monitor for BashArena
Track 2: Protocol Design Accessible
Write a Luthien policy that watches agent actions in BashArena and flags suspicious sysadmin commands before they run. Ctrl-Z showed resample protocols cut attack success from 58% to 7%. Can a proxy-level monitor do better?
Luthien provides: request interception, pattern matching, full audit logging
LLM-as-Judge Interrogation Protocol
Track 2: Protocol Design Moderate
Before the untrusted model's action runs, a trusted model reviews it and asks clarifying questions. Extend Luthien's built-in LLM-as-judge policy to add interrogation and measure the safety/usefulness tradeoff at different depths.
Luthien provides: SimpleLLMJudge policy (extend it), response interception, logging
Red Team Luthien's Policies
Track 3: Red Teams Accessible
Try to get harmful actions past Luthien's built-in defenses (command blocking, DLP, LLM judge). Document which evasion strategies work, which don't, and why. Build a dataset of adversarial transcripts.
Luthien provides: real policies to attack, full attack/defense audit trail
Multi-Policy Firewall for LinuxArena
Track 2: Protocol Design Advanced
Stack multiple Luthien policies (regex + DLP + LLM judge) into a layered defense. Test against LinuxArena tasks: does composing simple policies beat a single sophisticated one? Eligible for $1,000 Equistamp LinuxArena prize.
Luthien provides: multi-policy pipeline, per-policy logging, configurable enforcement
Steganography Detection via Proxy Logging
Track 3: Red Teams Advanced
Use Luthien's conversation logs to spot steganographic communication between colluding agents. Look for weird token distributions, formatting anomalies, and statistical signals that shouldn't be there.
Luthien provides: complete request/response capture, token-level logging
Code Review with Hidden Vulnerabilities
Track 1: Setting Design Moderate
An AI agent does code review (main task) while trying to approve code with hidden security vulnerabilities (side task). Luthien watches the review process and flags suspicious approvals.
Luthien provides: review interception, vulnerability pattern matching, audit trail
Policy Benchmark: DeSlop as Canary
Track 2: Protocol Design Accessible
Use Luthien's DeSlop policy (removes AI writing tics) as a simple canary to validate your evaluation pipeline. If you can measure DeSlop's effect reliably, swap in harder policies and trust the measurements. Good starter project.
Luthien provides: DeSlop ships built-in, trivially testable baseline
Trusted Editing via Proxy Rewrite
Track 2: Protocol Design Advanced
Luthien intercepts suspicious code suggestions and has a trusted model rewrite them before they reach the developer. Does the editor keep the code working while removing sabotage? Build it AND red-team it.
Luthien provides: response modification pipeline, two-model setup, before/after logging
