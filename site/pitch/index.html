Problem: Devs

Claude Code's Squandered Potential

Developers are frustrated: they can't let Claude Code run, so they babysit it instead.

Claude Code can run autonomously for 12 hours. In practice, humans babysit in 45-second increments because the risks of letting it run are too high: compliance violations, technical mistakes, security vulnerabilities, or simply failing to follow instructions.

12-hour autonomy: Claude Code documentation, Anthropic. 45-second babysitting cadence: observed across 20+ user discovery calls with developers using Claude Code 30+ hrs/week (Luthien internal research, Mar-Apr 2026).

Defensibility

Won't Anthropic build this?

1. They acknowledged our approach is the long-term solution
"The long-term solution probably looks more like something proxy-side" and "the space for control protocols is quite big." (Fabien Roger, Anthropic, Apr 1 2026)
2. Doesn't make sense for labs to build provider-agnostic tooling
The coding agent market is fragmenting fast: OpenCode, Codex, Cursor, OpenClaw. Anthropic won't build management tooling for competitors' clients.
3. We help drive Claude Code enterprise adoption
LLM gateways are already an enterprise infrastructure standard, and Claude Code is designed to work with them. Anthropic benefits from what we're building: it accelerates enterprise adoption of Claude Code.
4. If they do, great: defense in depth
More layers of safety is the goal. And we'll focus on Codex, Cursor, OpenCode, OpenClaw, and everything else they won't build for.
Fabien Roger quotes: from a private call with Anthropic's agent safety team, Apr 1, 2026. Shared with permission.
